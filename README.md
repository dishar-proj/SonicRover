# Sonic Rover
Contributors: Disha Rachur, Emerson Hall, Zane Bush, Garv Bhumbla, Navangana Jha
Components: Raspberry Pi, H-Bridge, Motors, Chebychev Filters, Python

User Manual: https://dr43632.wixsite.com/my-site-3

The main objective of this project is to design a car that can detect and move to multiple fixed audio frequencies. Each team received a kit of parts to build an audio car. The design should prioritize cost-effective solutions to minimize financial strain. The team is free to utilize parts they buy with their own budget, however, if any component burns or gets damaged, the team is responsible for providing and paying for a new one. 

The design team is composed of five members. Garv Bhumbla (EE) is responsible for the hardware, power supply, and wiring components.  Zane Bush (EE) is responsible for operating the RP4 and looking over hardware to check functionality. Emerson Hall (CSE) is responsible for writing and debugging code, and monitoring the ADC. Navangana Jha (CSE) is responsible for checking and modifying the code if issues arise, and helps with documentation. Disha Rachur (CSE) is responsible for task organization, documentation, and some wiring. 

This project is organized into three major checkpoints. The first checkpoint requires the car to travel along a straight line path, square path, and a figure eight path. The second checkpoint consists of the car detecting the C8 frequency, moving towards it, and stopping. The final checkpoint requires the car to detect and move towards the C8 frequency and then detect the and move towards the C6 frequency. The first checkpoint is the minimum for a C, the second checkpoint is the minimum required for a B, and the third checkpoint is the minimum for an A-. Each week, teams will be assigned a progress deliverable or a design deliverable to add and refine the functionality of the audio car. The weekly tasks ensure that the group is able to work collaboratively toward project goals.

In addition to technical development, the project emphasizes teamwork and collaboration. CATME surveys were conducted throughout the duration of the project to assess team dynamics, address team issues, and enhance team efficiency.
